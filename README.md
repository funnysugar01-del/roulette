# 발표자 순서 뽑기 웹앱 — 요구사항 명세서

## 1. 목적
교육생 발표 순서를 한 번에, 스릴있게 결정하는 웹앱.
커서AI 워크숍 실습 예제이자 실제 강의 도구로 활용.

## 2. 핵심 기능
- 이름 목록 입력 (줄바꿈으로 구분, 최소 2명 이상)
- 공개 연출 방식 선택 (토글, 3종 중 택1, 기본값: A. 회전판 룰렛)
  - **A. 회전판 룰렛**: 남은 후보들로 만든 원형 룰렛 휠이 돌다가 상단 포인터가 가리키는 이름으로 확정
  - **B. 텍스트 룰렛**: 이름이 빠르게 바뀌다가 딱 멈추며 확정 (슬롯머신 텍스트 효과)
  - **C. 세로 슬롯머신**: 이름들이 세로로 쌓인 띠가 위로 흐르다가 정답에서 멈추며 확정
- "순서 뽑기 시작" 버튼 클릭 → 1등부터 마지막 등수까지, 순위마다 선택한 연출 방식으로 빠르게 확정 공개되는 과정을 순차 반복
- 확정된 순위는 화면에 위→아래로 순서대로 쌓여서 표시
- 항목 등장(확정) 시마다 효과음, 전체 등장 완료 후 컨페티 효과
- "다시 뽑기" 버튼으로 재시작 가능

## 3. 기술 스택
- HTML / CSS / JavaScript 단일 파일 (index.html)
- 서버·DB 불필요 → 로컬 실행 또는 GitHub Pages 배포 가능
- 이유: 연속 애니메이션(슬롯머신, 순차 reveal, 컨페티)은 순수 JS가 Streamlit보다 자연스럽고 부드러움

## 4. 화면 설계

### 화면 1: 명단 입력
- 이름을 줄바꿈으로 입력하는 텍스트 영역
- "현재 인원: N명" 실시간 표시
- 공개 연출 방식 선택 토글: **A. 회전판 룰렛** / **B. 텍스트 룰렛** / **C. 세로 슬롯머신** (기본값: A)
- 최소 2명 이상이면 "순서 뽑기 시작" 버튼 활성화 (20명 다 채울 필요 없음)

### 화면 2: 룰렛 공개
- 상단에 "N / 전체인원 번째 순서 뽑는 중..." 진행 표시
- 선택한 연출 방식에 따라 순위마다 다음과 같이 확정:
  - A: 남은 후보로 구성된 원형 룰렛 휠이 회전하다 상단 포인터가 가리키는 이름에서 정지
  - B: 이름이 빠르게 바뀌다가 딱 멈추며 확정
  - C: 세로 띠가 위로 흐르다 정답 이름에서 정지
- 확정된 순위는 화면에 위→아래로 순서대로 쌓이며 표시, 확정될 때마다 효과음 재생
- 1등부터 마지막 등수까지 자동으로 반복 진행
- 전체 인원 공개 완료 후 컨페티 효과 실행, "다시 뽑기" 버튼 노출

### 디자인 톤앤매너
- 다크 배경 + 골드/레드 포인트 컬러
- 크고 굵은 임팩트 폰트
- 게임쇼/시상식 느낌
